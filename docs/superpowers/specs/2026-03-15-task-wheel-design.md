## 一、产品概述

### 1.1 产品定位

一款 iOS 应用，通过**同心圆轮盘拖拽交互**，让用户在 **2.5 秒内** 快速选定任务主题，效率比打字输入快 2 倍以上。

### 1.2 核心价值

| 维度 | 传统打字 | 同心圆轮盘 | 提升 |
|------|---------|-----------|------|
| **时间** | 5 秒 | 2.5 秒 | 2 倍 |
| **操作** | 双手键盘 | 单手拖拽 | 更便捷 |
| **认知** | 回忆 + 拼写 | 识别 + 选择 | 更轻量 |

### 1.3 目标用户

- 需要频繁记录/分类任务的用户
- 追求效率的极客用户
- 单手操作场景（通勤、排队等）

## 二、交互设计

### 2.1 主界面布局

```
┌─────────────────────────────┐
│         状态栏              │
├─────────────────────────────┤
│                             │
│    ╭─────────────────╮      │  ← 外圈：一级大类（8 个）
│   ╱    一级轮盘      ╲     │
│  │  ╭─────────────╮  │     │  ← 中圈：二级细分（5-7 个）
│  │ ╱  二级轮盘    ╲  │     │
│  │ │ ╭─────────╮ │  │     │  ← 内圈：具体任务（3-5 个）
│  │ │ │ 三级    │ │  │     │
│  │ │ │  写代码  │ │  │     │
│  │ │ ╰─────────╯ │  │     │
│  │ ╰─────────────╯  │     │
│   ╰─────────────────╯      │
│                             │
│  ← 边缘滑动返回             │
└─────────────────────────────┘
```

### 2.2 交互流程

```
用户操作          系统响应          累计时间
─────────────────────────────────────────────
1. 打开应用   →   显示一级轮盘      0ms
2. 手指滑动   →   轮盘惯性旋转      0-500ms
3. 选中项停留 →   高亮 + 轻微震动   500ms
4. 自动确认   →   放大进入下一级    700ms
5. 后台预加载 →   模型生成子主题    (并行)
6. 重复 2-5   →   2-3 级后完成      ≤2500ms
```

### 2.3 手势定义

| 手势 | 作用 | 技术实现 |
|------|------|---------|
| **水平滑动** | 旋转当前轮盘 | `UIPanGestureRecognizer` |
| **边缘→中心滑动** | 返回上一级 | `UIScreenEdgePanGestureRecognizer` |
| **双击中心** | 快速完成（跳过剩余层级） | `UITapGestureRecognizer` |
| **长按** | 显示主题说明/快捷编辑 | `UILongPressGestureRecognizer` |

### 2.4 确认机制

- **停留确认**：选中项在顶部（12 点钟方向）停留 **0.5 秒** 后自动确认
- **视觉反馈**：选中扇区高亮（绿色）+ 轻微震动（`UIImpactFeedbackGenerator`）
- **听觉反馈**：轻微"哒"声（可选，默认关闭）

## 五、性能优化

### 5.1 预加载策略

```
一级选择时 → 后台生成二级主题（并行）
二级选择时 → 后台生成三级主题 + 预渲染动画
三级选择时 → 直接显示结果
```

### 5.2 缓存层级

| 缓存类型 | 内容 | 容量 | 淘汰策略 |
|---------|------|------|---------|
| **L1 (内存)** | 当前轮盘数据 | - | 立即淘汰 |
| **L2 (NSCache)** | 常用主题树 | 50MB | LRU |
| **L3 (磁盘)** | 完整主题库 + 模型 | - | 手动更新 |

### 5.3 动画优化

- 使用 `CADisplayLink` 同步屏幕刷新（120Hz ProMotion 支持）
- Metal 渲染降低 CPU 负载
- 手势识别在独立线程，不阻塞 UI 主线程
- 使用 `UIViewPropertyAnimator` 实现可中断动画

## 九、风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 模型生成延迟 | 用户体验下降 | 预加载 + 缓存常用主题 |
| 惯性难以控制 | 选择效率低 | 调优阻尼参数，提供灵敏度设置 |
| 小屏幕适配困难 | 部分用户无法使用 | 最小支持 iPhone SE，提供缩放选项 |
| 电量消耗高 | 用户流失 | 优化 Metal 渲染，限制后台刷新 |
